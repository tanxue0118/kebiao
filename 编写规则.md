# 课程表编写规则

这个文件可以直接和课程表截图、文字版课表、教务系统导出的内容一起发给 AI，让 AI 按规则生成 `schedule.json`。也可以给同学手动照着写。

## 给 AI 的提示词模板

把下面这段复制给 AI，再附上你的课程表图片或文字：

```text
请根据我提供的课程表，生成一个课程表 JSON。

要求：
1. 只输出 JSON，不要输出解释。
2. 根对象包含 semesterStart、semesterWeeks、timeSlots、courses。
3. semesterStart 是开学日期，格式 YYYY-MM-DD；如果我没有提供，请先根据我说的“第几周”推算，推算不了就用空字符串。
4. semesterWeeks 是本学期周数，默认 16。
5. dayOfWeek 用数字表示：周一=1，周二=2，周三=3，周四=4，周五=5，周六=6，周日=7。
6. weeks 用字符串写法，例如 "1-16"、"1-8,10,12-16"、"17"。
7. 单双周写 weekParity：每周=all，单周=odd，双周=even。
8. 普通课程填写 startSection 和 endSection。
9. 如果课程没有节次，只有具体时间，写 customTimeEnabled: true，并填写 startTime、endTime。
10. 每门课都要有 id、name、teacher、location、dayOfWeek、weeks、weekParity。
11. id 用英文或拼音加短横线，不能重复。
12. 不确定的老师或地点用空字符串。
13. timeSlots 按下面默认时间填写，除非我提供了不同作息时间。

默认 timeSlots：
1 08:00-08:45
2 08:55-09:40
3 10:00-10:45
4 10:55-11:40
5 14:30-15:15
6 15:25-16:10
7 16:20-17:05
8 17:15-18:00
9 18:40-19:25
10 19:35-20:20
11 20:30-21:15
12 21:25-22:10
```

    { "number": 2, "startTime": "08:55", "endTime": "09:40" },
    { "number": 3, "startTime": "10:00", "endTime": "10:45" },
    { "number": 4, "startTime": "10:55", "endTime": "11:40" },
    { "number": 5, "startTime": "14:30", "endTime": "15:15" },
    { "number": 6, "startTime": "15:25", "endTime": "16:10" },
    { "number": 7, "startTime": "16:20", "endTime": "17:05" },
    { "number": 8, "startTime": "17:15", "endTime": "18:00" },
    { "number": 9, "startTime": "18:40", "endTime": "19:25" },
    { "number": 10, "startTime": "19:35", "endTime": "20:20" },

## 字段规则

### 学期字段

- `semesterStart`：开学日期，格式 `YYYY-MM-DD`。
- `semesterWeeks`：本学期周数，一般是 `16`、`18` 或 `20`。

如果你只知道“某一天是第几周”，可以让 AI 反推开学日期。例如：

```text
2026-06-15 这一周是第 16 周，请反推 semesterStart。
```

### 节次时间

每个 `timeSlots` 项表示一节课：

```json
{ "number": 1, "startTime": "08:00", "endTime": "08:45" }
```

规则：

- `number` 是第几节。
- `startTime` 和 `endTime` 必须是 `HH:mm`。
- 如果学校作息不同，直接改这里。

### 课程字段

普通课程：

```json
{
  "id": "college-english",
  "name": "大学英语",
  "teacher": "秦老师",
  "location": "3-206",
  "dayOfWeek": 3,
  "weeks": "1-16",
  "weekParity": "all",
  "startSection": 1,
  "endSection": 2
}
```

单双周课程：

```json
{
  "id": "physics-lab-odd",
  "name": "大学物理实验",
  "teacher": "李老师",
  "location": "实验楼 206",
  "dayOfWeek": 4,
  "weeks": "1-16",
  "weekParity": "odd",
  "startSection": 7,
  "endSection": 8
}
```

考试或临时课程：

```json
{
  "id": "exam-c-language",
  "name": "C 语言考试",
  "teacher": "",
  "location": "4-102",
  "dayOfWeek": 1,
  "weeks": "17",
  "weekParity": "all",
  "customTimeEnabled": true,
  "startTime": "15:00",
  "endTime": "17:00"
}
```

## 周次写法

推荐使用字符串：

```json
"weeks": "1-16"
```

也可以写多个范围：

```json
"weeks": "1-8,10,12-16"
```

只上一周：

```json
"weeks": "17"
```

也兼容数组：

```json
"weeks": [1, 2, 3, 4, 5]
```

## 星期写法

`dayOfWeek` 必须是数字：

```text
周一 = 1
周二 = 2
周三 = 3
周四 = 4
周五 = 5
周六 = 6
周日 = 7
```

## 单双周写法

```text
每周 = "all"
单周 = "odd"
双周 = "even"
```

示例：

```json
"weekParity": "even"
```

## 常见转换规则

- “1-2 节”写成 `startSection: 1`、`endSection: 2`。
- “第 5、6 节”写成 `startSection: 5`、`endSection: 6`。
- “周一 1-2 节”写成 `dayOfWeek: 1`、`startSection: 1`、`endSection: 2`。
- “单周上课”写成 `weekParity: "odd"`。
- “双周上课”写成 `weekParity: "even"`。
- “1-8 周、10 周、12-16 周”写成 `weeks: "1-8,10,12-16"`。
- 老师未知写 `teacher: ""`。
- 地点未知写 `location: ""`。

## 检查清单

生成 JSON 后，请检查：

- JSON 没有注释、没有多余逗号。
- 每门课都有唯一 `id`。
- `dayOfWeek` 是 1 到 7。
- `weeks` 不要写“第 1-16 周”，直接写 `"1-16"`。
- `weekParity` 只能是 `all`、`odd`、`even`。
- 普通课程有 `startSection` 和 `endSection`。
- 临时考试如果没有节次，要写 `customTimeEnabled: true`、`startTime`、`endTime`。
- 时间格式是 `08:00`，不要写 `8:00`。
